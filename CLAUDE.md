# ニュース自動配信アプリ - CLAUDE.md

最終更新日：2026年4月18日

---

## 1. プロジェクト概要

特定キーワードに関するニュース記事を毎日自動収集し、登録済みメールアドレスへ配信するWebアプリケーション。
ニュース記事の手動検索にかかる時間を削減することを目的とする。

---

## 2. アプリのゴール（WHO / WHAT / HOW）

| 項目 | 内容 |
|------|------|
| WHO  | 職場の同僚5名程度（管理者1名含む）|
| WHAT | 特定キーワードに合致したニュース記事を毎日受け取る |
| HOW  | 毎朝8時に記事タイトル＋URLをメールで自動配信する |

---

## 3. ターゲットユーザー詳細

| ユーザー種別 | 人数 | 権限 |
|-------------|------|------|
| 管理者 | 1名 | 管理者画面へのログイン・キーワード管理・配信先管理 |
| 一般ユーザー | 4名程度 | メール受信のみ |

---

## 4. 主要機能

### MVP（テスト版）- 優先実装

| 機能 | 詳細 |
|------|------|
| 記事収集 | Yahoo ニュースからキーワード合致記事を取得 |
| メール配信 | 毎朝8時に全登録アドレスへ一斉送信 |
| メール形式 | 記事タイトル＋URLの一覧形式 |
| 記事なし通知 | 該当記事が0件の日も「該当記事なし」とメール送信 |
| 管理者ログイン | ID・パスワード認証 |
| キーワード管理 | 管理者画面からキーワードを追加・変更・削除 |
| 配信先管理 | 管理者画面からメールアドレスを追加・削除 |

### 将来機能（完成版）

| 機能 | 詳細 |
|------|------|
| 対象サイト拡張 | 日経電子版・朝日新聞デジタルを追加 |
| 配信時間変更 | 管理者画面から配信時間を変更できる |

> **注意：** 日経電子版・朝日新聞デジタルは有料コンテンツが多く、記事取得に制限がかかる場合がある。完成版開発時に取得可能範囲を確認すること。

---

## 5. UI/UX 方針

- シンプルで直感的な管理画面を優先する
- 管理者が迷わず操作できる最小限の画面構成にする

### 画面構成

| 画面 | 役割 |
|------|------|
| 管理者ログイン画面 | ID・パスワードで管理者がログイン |
| 管理者ダッシュボード | キーワード・配信先メールアドレスの一覧・追加・削除 |

## 6. 技術仕様

| 項目 | 採用技術 |
|------|---------|
| 開発言語 | Python 3.11 |
| Webフレームワーク | Flask |
| データベース | SQLite |
| 記事収集 | BeautifulSoup4 + requests |
| メール送信 | Gmail（smtplib）|
| スケジューラ | APScheduler |
| 実行環境 | クラウドサーバー（Render）|
| 開発環境 | GitHub Codespaces（ブラウザ上で開発）|
| バージョン管理 | GitHub |

### フォルダ構成（予定）

```
news-app/
├── app.py              # アプリ本体
├── scraper.py          # 記事収集
├── mailer.py           # メール送信
├── scheduler.py        # 定期実行
├── models.py           # データベース定義
├── templates/          # HTML画面
│   ├── login.html
│   └── dashboard.html
├── requirements.txt    # 使用ライブラリ一覧
└── .env                # 環境変数（Git管理外）
```

## 8. 成功基準

| 基準 | 内容 |
|------|------|
| 配信安定性 | 毎朝8時に確実にメールが届く |
| 記事精度 | 登録キーワードに合致した記事のみ配信される |
| 管理操作 | 管理者が5分以内にキーワード・配信先を変更できる |
| 無記事対応 | 該当記事なしの日も「該当記事なし」メールが届く |
